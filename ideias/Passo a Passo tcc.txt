Capítulo 2 — Referencial Teórico (Fundamentação)
O que é
Aqui você não fala do seu sistema ainda.
Você explica os conceitos teóricos que sustentam o seu trabalho.
É como se fosse:

	“Antes de explicar meu sistema, vou explicar os conceitos da área”.

O que você deve falar no seu caso
Para seu sistema de estoque e vendas, normalmente aparecem esses tópicos:

2.1 Sistemas de Informação
Explicar o que são sistemas de informação.
Exemplo:
- conceito
- importância nas empresas
- automação de processos

2.2 Controle de Estoque
Explicar:
- o que é estoque
- importância do controle
- problemas de controle manual

2.3 Banco de Dados
Explicar:
- o que é banco de dados
- importância no armazenamento de dados
- modelo relacional

2.4 Engenharia de Software
Explicar:
- desenvolvimento de software
- organização de sistemas
- ciclo de vida do software

2.5 Arquitetura em Camadas
Aqui entra seu sistema.
Explicar:
- o que é arquitetura em camadas
- vantagens
- separação de responsabilidades

2.6 Trabalhos Relacionados
Aqui você mostra:
outros sistemas ou pesquisas parecidas.
Exemplo:
- sistemas de controle de estoque
- sistemas de vendas
- sistemas para pequenos comércios

Objetivo do Capítulo 2
Mostrar que você estudou o assunto antes de desenvolver o sistema.
Normalmente tem 8 a 15 páginas.

Capítulo 3 — Metodologia
Aqui você explica:
COMO você fez o trabalho Não é teoria. É o processo do projeto.
O que entra aqui
Tipo de pesquisa
Exemplo:
- pesquisa aplicada
- abordagem qualitativa
- objetivo exploratório
Exemplo de texto:
	Este trabalho caracteriza-se como uma pesquisa aplicada, pois busca desenvolver uma solução prática para o controle de estoque e vendas em uma cantina escolar.

Método de desenvolvimento
Você explica:
- levantamento de requisitos
- modelagem do banco
- desenvolvimento do sistema
- testes

Ferramentas utilizadas
Exemplo do seu projeto:
- C#
- .NET
- SQL Server
- Visual Studio

Metodologia de desenvolvimento
Pode falar:
- modelo cascata
- prototipação
Exemplo:
	O desenvolvimento seguiu um modelo incremental, iniciando com levantamento de requisitos, seguido pela modelagem do sistema e implementação das funcionalidades.

Normalmente tem 4 a 6 páginas.
